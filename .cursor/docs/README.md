## üìù Documentation Categories Explained

### Guides vs. Architecture
- **Guides** are step-by-step instructions for setup and configuration
- **Architecture** documents system design, components, and data flow

### Planning vs. Analysis
- **Planning** documents are forward-looking: roadmaps, design documents, implementation plans
- **Analysis** documents are analytical: competitive research, thought pieces, comparative studies

### Protocols vs. Philosophy
- **Protocols** are operational procedures: how to navigate, how to operate
- **Philosophy** is conceptual exploration: why we build this way, semantic space mapping

### Status
- **Status** tracks current state: what's done, what's pending, what's next
